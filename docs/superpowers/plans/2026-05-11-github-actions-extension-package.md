# GitHub Actions Extension Package Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add GitHub Actions automation that validates, builds, zips, uploads, and tag-releases the AtWebPilot Chrome extension.

**Architecture:** A single workflow file owns CI packaging. It uses Corepack-managed pnpm, existing package scripts, a shell zip step rooted in `dist/`, artifact upload for every run, and GitHub Release upload for `v*` tag runs.

**Tech Stack:** GitHub Actions, Node 20, pnpm via Corepack, Vite/CRX build output, POSIX shell, zip.

---

## File Structure

- Create `.github/workflows/build-extension.yml`: CI workflow for validation, packaging, artifact upload, and tag release upload.
- Modify `README.md`: document where to download build artifacts and how to create a release package.

- [ ] **Step 1: Create workflow directory**

Run:
```bash

```

Expected: `.github/workflows` exists.

- [ ] **Step 2: Create workflow file**

Write this exact content to `.github/workflows/build-extension.yml`:

```yaml
name: Build Extension

on:
  push:
    branches:
      - "**"
    tags:
      - "v*"
  pull_request:
  workflow_dispatch:

permissions:
  contents: write

jobs:
  package:
    name: Typecheck, Test, Build, and Package
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Enable pnpm
        run: |
          corepack enable
          corepack prepare pnpm@9 --activate

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Typecheck
        run: pnpm typecheck

      - name: Test
        run: pnpm test

      - name: Build extension
        run: pnpm build

      - name: Read package version
        id: package
        run: |
          version=$(node -p "require('./package.json').version")
          echo "version=$version" >> "$GITHUB_OUTPUT"
          echo "zip_name=atwebpilot-$version.zip" >> "$GITHUB_OUTPUT"

      - name: Package dist
        run: |
          cd dist
          zip -r "../${{ steps.package.outputs.zip_name }}" .

      - name: Upload packaged extension
        uses: actions/upload-artifact@v4
        with:
          name: atwebpilot-extension-${{ steps.package.outputs.version }}
          path: ${{ steps.package.outputs.zip_name }}
          if-no-files-found: error

      - name: Publish GitHub Release
        if: startsWith(github.ref, 'refs/tags/v')
        env:
          GH_TOKEN: ${{ github.token }}
          ZIP_NAME: ${{ steps.package.outputs.zip_name }}
        run: |
          tag="${GITHUB_REF_NAME}"
          if gh release view "$tag" >/dev/null 2>&1; then
            gh release upload "$tag" "$ZIP_NAME" --clobber
          else
            gh release create "$tag" "$ZIP_NAME" --title "$tag" --generate-notes
          fi
```

````markdown
## GitHub Actions 打包

仓库包含 `.github/workflows/build-extension.yml` 自动打包流程：

- `push` / `pull_request` / 手动运行会执行 `pnpm typecheck`、`pnpm test`、`pnpm build`，并上传 `atwebpilot-<version>.zip` artifact。
- 推送 `v*` tag（例如 `v0.0.1`）会在通过检查后创建 GitHub Release，并上传同一个 zip。
- zip 内容来自 `dist/` 内部，`manifest.json` 位于压缩包根目录，可直接用于 Chrome 扩展加载或发布前检查。

发布示例：

```bash
git tag v0.0.1
git push origin v0.0.1
```
````
